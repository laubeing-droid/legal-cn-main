# 中国化适配原则

> 本仓库对上游 Anthropic 法律 AI 做了哪些中国化改造、遵循什么原则、不做什么。

---

## 一、什么是"中国化适配"

不是翻译。是**逐层替换美国法的法律推理结构**。

| 层级 | 美国法残留 | 中国化处理 |
|:-----|:----------|:----------|
| 术语 | "deposition""discovery""consideration" | 阻断或替换为调查取证准备/证据交换/对价不存在 |
| 法条 | 美国 UCC/Restatement | 民法典/公司法/劳动法等 22 部中国法律 |
| 工作流 | IRAC 判例推理结构 | 请求权基础分析法（目标→法条→构成要件→效果） |
| 工具 | 美国法律数据库 MCP | 北大法宝/威科先行等中国工具 |
| 思维 | 对抗制 + 判例法 | 成文法 + 审查/合规导向 |

---

## 二、适配原则

### 原则 1：无对应制度 → 阻断

中国法不存在这个制度 → 直接阻断，不给用户"这个制度在中国可以用"的幻觉。

示例：deposition、privilege log、habeas corpus、statute of frauds

### 原则 2：有对应制度但不同 → 替换

中国法有类似制度但运作逻辑不同 → 替换为中国法等效概念 + 解释差异。

示例：plea bargaining → 认罪认罚从宽、discovery → 证据交换

### 原则 3：有对应制度且相同 → 保留

中美国法概念实质一致 → 保留原术语，仅替换引法依据。

### 原则 4：中国法独有制度 → 新增

中国法有而美国法没有的制度 → 新增专门技能或 prompt。

示例：经济性裁员（劳动法第41条）、认罪认罚从宽（刑诉法第15条）

## 三、改造范围

### 已完成的改造

| 维度 | 深度 |
|:-----|:----:|
| 子技能重写 | 5 个全量重写（调查取证准备/证据保全与留存/司法协查响应/律师函生成/诉讼留证） |
| 子技能删除 | 1 个（privilege-log-review） |
| 子技能改名 | 1 个（cease-desist → 律师函生成） |
| CLAUDE.md | 12 领域全中文化 |
| MCP 连接器 | 12 领域全部替换为中国工具链 |
| 法条引用 | 0 → 22 部官方 PDF |
| 概念阻断 | 22 → 29 个美国法概念清单 |
| 12 领域推理模板 | 自研（合同/诉讼/劳动/数据/公司/IP/产品/监管/AI/教育/法援/构建器） |

### 不改的内容

- Anthropic 的目录结构（保持兼容）
- 技能系统的元数据格式（保持可识别）
- SKILL.md 的触发条件和参数定义（保持功能）

## 四、护栏层设计

### 阻断清单（blocking-list.md）

29 个中国法不存在或不兼容的美国法概念，AI 输出时必须阻断或替换。

分为 4 类处理：
- ❌ 整行删除：中国完全没有的制度
- 替换：有中国等效制度
- 阻断 + 替换：部分替代
- 阻断 + 法域隔离：跨境引用

### 元规则（meta-rules.md）

约束 AI 推理框架的全局规则：
- Rule A：拦截即阻断
- Rule B：不明则问
- Rule C：阻断必替代
- Rule D：法域隔离
- Rule E：成文法优先

### 香港桥梁（hk-bridge.md）

处理涉及香港的跨境法律问题，防止中国法和香港普通法概念混淆。

## 五、质量保障

- 对抗测试集（15 用例）验证护栏是否生效
- verify.ps1 检查护栏文件完整性
- GitHub Actions 自动扫描美国法术语残留
- diff-tool 参考窗口模式防止上游变更污染

---

## 六、适配不是一次性的

中国法律持续更新（新司法解释、新指导案例、法规修订），本仓库的适配工作也需要持续跟进。参见 `UPSTREAM_DIFF_POLICY.md` 了解上游变更处理流程。
