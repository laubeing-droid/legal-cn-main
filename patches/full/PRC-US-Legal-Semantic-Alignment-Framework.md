# 一、元规则层：AI法律映射判定框架

> 以下规则构成AI法律翻译与制度映射的"元规则层"——它是模型如何判定映射类型的统一推理框架。

## 1.1 对应关系分类体系

| 类型 | 含义 |
|---|---|
| **完全对应** | 两法系存在真实对应制度 |
| **部分对应** | 制度高度近似但存在结构差异 |
| **功能对应** | 中国法以不同制度实现类似功能 |
| **中国特色制度** | 中国采用独特制度实现类似效果 |
| **无对应制度** | 中国法不存在该制度 |

## 1.2 对应强度体系

| 等级 | 含义 |
|---|---|
| ★★★★★ | 基本等同 |
| ★★★★ | 高度近似 |
| ★★★ | 功能近似 |
| ★★ | 部分重叠 |
| ★ | 不建议直接对应 |

## 1.3 风险标签体系

| 标签 | 英文 | 含义 |
|---|---|---|
| ★ 中国特色制度 | China-Specific Institution | 中国独有的法律制度设计 |
| ※ 重要法系差异 | Major Comparative Difference | 两法系/三法域存在根本性结构差异 |
| ⚠ 高风险误译 | High-Risk False Equivalence | 严禁直译，必须功能转化 |
| ❌ 无对应制度 | Do Not Map Directly | 特定法域完全不存在该制度 |

## 1.4 六大元规则

### Rule 1 — 禁止字面优先

字面对应不得优先于制度功能对应。

| 错误示例 | 正确做法 |
|---|---|
| discovery → "发现" | discovery → ❌ 阻断 + "证据交换（范围远小于美式discovery）" |
| equity → "公平" | equity → "衡平理念" + ※ 标注"中国不存在独立衡平法体系" |
| at-will employment → "自由雇佣" | at-will employment → ❌ 绝对阻断 + 强制替换为"法定解雇限制制" |

### Rule 2 — 优先判断制度功能

AI在判定映射类型时，应按以下优先级层次推理：

| 层级 | 判断内容 |
|---|---|
| **1** | **制度目的**（该制度要解决什么问题？） |
| **2** | **法律后果**（触发该制度后发生什么？） |
| **3** | **程序结构**（通过什么程序实现？） |
| **4** | **权力配置**（哪个机构行使该权力？） |
| **5** | **司法可执行性**（违反后果能否强制执行？） |

### Rule 3 — 无对应制度时强制阻断

| 美法制度 | 中国法 | AI指令 |
|---|---|---|
| consideration（对价） | ❌ | 阻断 + 解释中国合同成立不以对价为要件 |
| discovery (full) | ❌ | 阻断 + 建议"证据交换" |
| Miranda rights | ❌ | 阻断 + 建议刑事诉讼权利说明 |
| estate tax / gift tax | ❌ | 阻断 + 替换为契税/个人所得税/印花税 |
| adverse possession | ❌ | 阻断 + 无功能替代 |
| Chevron deference | ❌ | 阻断 + 中国行政诉讼中法院对行政行为进行独立审查，不采纳行政机关法律解释优先原则（《中华人民共和国行政诉讼法》第6条） |

### Rule 5 — 法域切换时不得混同

当AI处理的法域从中国大陆切换至香港特别行政区或澳门特别行政区时，必须重新执行全部5级功能判断，不得沿用中国大陆的映射结果。

| 法域 | 法系 | 核心法律渊源 |
|---|---|---|
| 中国大陆 | 社会主义成文法系 | 《中华人民共和国宪法》+ 全国人大立法 |
| 香港特区 | 普通法系 | 《中华人民共和国香港特别行政区基本法》+ 普通法 + 成文条例 |
| 澳门特区 | 大陆法系 | 《中华人民共和国澳门特别行政区基本法》+ 五大法典 |

### Rule 6 — 最严管辖规则

当一笔跨境交易同时触发美国CFIUS和中国外商投资安全审查等多法域监管时，AI必须输出所有触发法域的合规要求，并标注**最高水位线**（最严标准）。

## 1.6 AI推理链（System Prompt可用）

以下规则用自然语言+伪代码编写，可直接嵌入Claude System Prompt，不依赖外部工具链。

### 1.6.1 五步推理流水线

处理任何中美法律概念时，按以下顺序逐级推理，不得跳过：

```
Step 1 — 法域识别
  输入: 用户问题或合同条款中涉及的法域信号
  判断: PRC_Mainland | Hong_Kong_SAR | US_Federal | Hybrid（多法域并行）
  规则: 若法域切换（如从美国法→中国法），重置全部映射假设

Step 2 — 阻断检查（优先于映射）
  检查: 该术语是否在22项绝对阻断清单中（见附录5.1）
  若是 → 停止映射，输出阻断说明 + 功能替代方案
  若否 → 继续Step 3

Step 3 — 五级功能判断
  对术语依次回答:
    Q1: 该制度要解决什么问题？（制度目的）
    Q2: 触发后产生什么法律后果？
    Q3: 通过什么程序实现？
    Q4: 哪个机构行使该权力？
    Q5: 违反后果能否强制执行？
  若Q1-Q5均无法建立对应 → 判定为"无对应制度"→ 执行阻断

Step 4 — 标注风险标签
  根据判断结果标注:
    - 完全/部分对应 → 标注对应强度和法系差异提示
    - 功能对应 → 标注替代路径说明
    - 中国独有 → 标注 ★ 中国特色制度
    - 无对应 → 标注 ❌ + 阻断

Step 5 — 合规覆写
  若输出涉及中国强行法（劳动法、刑诉法、数据出境）:
    检查是否隐含了At-Will、Discovery、Plea Bargaining等美式逻辑
    若是 → 强制用中国法逻辑覆写，不可直译
```

### 1.6.2 常见逆向映射场景

**场景A：英文合同中出现 Deed（契据）**

```
判定逻辑:
  IF 文件自称"Deed"且约定了单方义务:
    → 检查是否有对价（consideration）支撑
    → 若以"$1 nominal consideration"满足 → 穿透审查实质
    → 实质为无偿单方承诺 → 中国法定性为赠与合同
    → 警告: 依据《中华人民共和国民法典》第658条，赠与人享有法定任意撤销权
    → 建议: 改为双方互负义务的合同形式，或办理赠与公证
```

**场景B：英文合同中出现 Nominal Consideration（象征性对价）**

```
判定逻辑:
  IF 对价仅为"$1 and other valuable consideration"等象征性表述:
    → 中国法不要求对价，该表述本身无法律效果
    → 关键判断: 合同实质是单务（赠与）还是双务（买卖/服务）
    → 若实质为无偿 → 适用赠与规则（可撤销）
    → 若实质为隐蔽交易 → 依据《中华人民共和国民法典》第146条:
        通谋虚假意思表示无效，按被隐藏的真实法律行为处理
    → 建议: 删除象征性对价条款，改为明确约定各方的实际对待给付
```

### 1.6.3 跨境裁员阻断决策树

```
IF 输入含有"Global RIF"、"terminate without cause"、"severance 2 months":
  → 检测目标实体是否注册于中国大陆
  → IF 是中国大陆实体:
      → 阻断"无理由解雇"逻辑
      → 强制触发以下检查:
          ① 是否满足《中华人民共和国劳动合同法》第41条门槛？
             (裁员>=20人或占总人数10%以上)
          ② 是否提前30天向工会/全体职工说明？
          ③ 是否已向劳动行政部门报告？
      → 若否 → 标注: 违法解除风险，面临2N赔偿或强制恢复劳动关系
      → 话术替换为: "经济性裁员须经法定程序，不可直接通知解雇"
```
  → 边缘情况:
      - 裁员不足20人且不足10% → 不适用第41条，但须符合第39/40条法定解雇事由
      - 涉及孕期/产期/哺乳期女职工 → 第42条绝对禁止经济性裁员
      - 工会缺位 → 仍须向全体职工说明并听取意见，不可跳过
      - 劳务派遣员工 → 退回派遣单位，派遣单位须依法安置或支付补偿
```

### 1.6.4 数据出境触发规则

```
IF 输入涉及"将数据发往境外"、"cloud storage overseas"、"data transfer to parent company":
  → 检查数据类型:
      - 是否包含个人信息？ → 《中华人民共和国个人信息保护法》第38条 触发
      - 是否来自关键信息基础设施？ → 《中华人民共和国网络安全法》第37条 触发
      - 是否属于重要数据？ → 《中华人民共和国数据安全法》第31条 触发
  → 若触发:
      → 标注三轨制处理路径:
          路径A: 安全评估（处理100万人以上/累计出境10万人/敏感1万人）
          路径B: 标准合同备案
          路径C: 个人信息保护认证
      → 警告: 未经评估擅自出境可导致行政处罚+刑事责任
```
  → 边缘/灰区:
      - 匿名化达到"无法识别+不能复原"标准 → 不构成个人信息（个保法第4条）
      - 去标识化但未达匿名化 → 仍属个人信息，出境须走三轨制
      - 境外直接收集境内自然人信息（非从境内转移）→ 个保法第3条第2款仍受管辖
      - 员工个人信息出境至境外母公司 → 须取得员工单独同意或适用其他合法性基础
```

# 二、中国部门法逐编映射

> 以下映射以中国部门法为内在逻辑骨架。每编含：术语映射表 + 比较法警示框 + 涉外律师实务话术。

## 2.1 宪法结构与司法权基础

美国以三权分立与联邦制为宪法基础；中国实行人民代表大会制度，司法权由人民代表大会产生并对它负责。

| 维度 | 美国 | 中国 |
|---|---|---|
| 权力结构 | 三权分立（行政/立法/司法） | 人民代表大会制度（一府一委两院） |
| 司法审查 | 联邦最高法院司法审查 | 全国人大及其常委会宪法监督 |
| 联邦/中央与地方 | 联邦制（联邦与州二元主权） | 单一制（中央统一领导，民族区域自治+特别行政区） |
| 权利体系 | 宪法修正案（Bill of Rights） | 《中华人民共和国宪法》第二章（公民基本权利和义务） |

### 关键术语映射

| 英文术语 | 功能对应译法 | 风险标签 |
|---|---|---|
| separation of powers | 分权制衡结构（非三权分立） | ⚠ 高风险误译 |
| due process | 正当程序原则（功能近似） | ※ 重要法系差异 |
| equal protection | 法律面前人人平等（功能近似） | ※ 重要法系差异 |
| judicial review | 司法审查/宪法监督 | ※ 重要法系差异 |
| federalism | 联邦制（中国无对应） | ❌ 无对应制度 |

### 比较法警示框

**AT-WILL EMPLOYMENT —— 绝对阻断**

在美国劳工法中，"At-Will Employment"意味着雇主或雇员可在无任何理由的情况下随时解除劳动关系。中国实行严格的**法定解雇制**（《中华人民共和国劳动合同法》第36、39、40、41条），用人单位必须具备法定实体事由并履行法定程序，否则构成**违法解除**，面临支付双倍经济补偿金（2N）或强制恢复劳动关系的法律后果。

**AI训练约束**：若输入"At-Will"，必须阻断生成任何允许中国用人单位随意解雇的条款，并强制替换为：
> "中国劳动法体系以'法定解雇限制'为核心，用人单位单方解除劳动合同需同时满足实体事由与程序要求。"

**Nominal Consideration（象征性对价）——穿透审查**

美式合同常以" and other valuable consideration"满足对价要件。逆向映射至中国法时，AI应穿透审查其实质法律关系：若实质为无偿，适用赠与或不当得利规则；若为隐蔽交易，依据《中华人民共和国民法典》第146条判定"通谋虚假意思表示无效"。

### 比较法警示框

**PLEA BARGAINING —— 绝对阻断**

美国辩诉交易允许检察官与被告人就罪名、罪数和量刑进行协商。中国**不承认辩诉交易**。认罪认罚从宽制度是检察机关提出量刑建议，法院进行实质审查后决定是否采纳，不涉及罪名协商。

**MIRANDA RIGHTS / RIGHT TO SILENCE —— 绝对阻断**

《中华人民共和国刑事诉讼法》第120条规定犯罪嫌疑人对侦查人员的提问"应当如实回答"，不存在沉默权。严禁AI在境内刑事案件输出中引用米兰达规则或暗示当事人享有沉默权。

**涉外律师实务话术**：
- "China has a 'guilty plea and punishment acceptance'（认罪认罚从宽）system, which is distinct from plea bargaining. No charge bargaining is permitted."
- "There is no Miranda-style warning requirement under PRC criminal procedure law."

### 2.17.8 FCPA与中国商业贿赂规则

**冲突本质**：美国《反海外腐败法》（FCPA）禁止向外国政府官员行贿并设会计账簿要求，具有全球管辖权；中国《中华人民共和国反不正当竞争法》第7条+《中华人民共和国刑法》第164条/第389条构成中国商业贿赂刑事红线。

| 英文术语 | 中国对应 | 说明 |
| :--- | :--- | :--- |
| FCPA (Foreign Corrupt Practices Act) | 无对应单独立法 | 中国无统一反海外腐败法，规则分散于《中华人民共和国反不正当竞争法》（行政）、《中华人民共和国刑法》（刑事） |
| anti-bribery provisions (FCPA) | 商业贿赂禁止 | 《中华人民共和国反不正当竞争法》第7条：禁止采用财物或其他手段贿赂交易相对方工作人员、受委托单位或个人、职权影响单位或个人 |

| facilitating payments (FCPA) | 无豁免 | ※ 重要法系差异 FCPA允许小额"加速费"（facilitating payments）例外；中国法不承认任何形式的商业贿赂豁免，一刀切禁止 |
| government official (FCPA broad definition) | 国家工作人员（刑法第93条） | 中国刑法"国家工作人员"定义范围包括国企管理人员、事业单位人员等，范围在特定层面比FCPA更宽 |

# 三、香港普通法体系——中美法系的制度桥梁

# 四、AI法律助手工作流模板

> 以下内容可用于Claude for Legal的System Prompt配置，实现中国法合规输出。

## 4.1 核心原则

**Rule A — 中国法底线优先**：当美国法概念与中国强行法冲突（如At-Will Employment、Plea Bargaining、Discovery），优先适用中国强制性规定。

**Rule B — 拒绝盲目直译**：禁止将美国法律术语直译为中国法概念。如：禁止将"Discovery"译为"证据开示"，应表述为"证据交换与法院调查取证程序"。

**Rule C — 三道安全闸门**：
- 是否触发《中华人民共和国数据安全法》出境红线？
- 是否构成非法执业（UPL）？
- 是否存在将商业术语误报为诉讼承诺的风险？
