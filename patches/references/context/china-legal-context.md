# WorkBuddy 中国法律语境总则

本参考文件适用于本仓库所有从 `claude-for-legal` 转换而来的 WorkBuddy skill。除非用户明确指定其他法域，默认工作语境为**中国大陆**；香港、澳门、台湾以及境外法域需单独确认。

## 优先规则

- 把下游 skill 中的美国州法、Delaware、ABA、CourtListener、DMCA、subpoena、deposition、attorney work product 等内容视为原始模板遗留项；遇到冲突时，以本文件和各模块 `references/china-context.md` 为准。
- 不作最终法律意见、法律结论或诉讼策略承诺。输出定位为“法务/律师审阅草稿”“内部合规分析”或“待执业律师确认的研究笔记”。
- 涉及法律、司法解释、部门规章、监管规则、国家标准、地方规则、裁判规则或有效日期时，必须要求核验现行文本；不能核验时标注 `[需核验]`。
- 中文输出优先。合同、函件和面向业务方的摘要使用简体中文；涉外交易可按用户要求双语输出。
- 日期使用 `YYYY-MM-DD` 或中文年月日；金额默认人民币，另有币种时明确标注。

## 法源和引用层级

优先使用官方来源或团队已授权数据库。常用来源：

## 中国语境下的审查姿态

- 合同审查：以《民法典》合同编、担保制度、格式条款、违约责任、解除、不可抗力、争议解决、数据/IP/保密条款为主线。
- 数据合规：区分个人信息、敏感个人信息、重要数据、核心数据、网络数据；关注告知同意、单独同意、共同处理/委托处理/向第三方提供、自动化决策、未成年人、保存期限、出境机制和安全事件。
- 公司治理：区分有限责任公司、股份有限公司、外商投资企业、上市公司/非上市公众公司；注意章程、股东会/董事会职权、法定代表人、董监高责任、关联交易和信息披露。
- 劳动用工：先识别劳动关系、劳务/承揽/合作、新就业形态；解除终止、调岗调薪、竞业限制、加班工时、休假、社保、公积金和员工手册民主程序需特别谨慎。
- 争议解决：优先确认法院/仲裁管辖、诉讼时效/仲裁时效、保全、证据规则、电子数据、举证责任、执行可行性。美国 discovery/deposition/subpoena 仅作比较法概念，不作默认流程。
- 知识产权：商标和专利以 CNIPA、法院/行政执法和平台投诉规则为主；著作权、商业秘密和不正当竞争需与《民法典》《反不正当竞争法》《著作权法》等衔接。
- 监管沟通：对外提交、监管问询回复、公开评论、律师函、诉讼/仲裁文件、董事会/股东会正式文件均需人工审阅并走发送/提交确认门。

## 输出标记

- 面向内部法务/律师：`保密 / 内部法律分析 / 律师工作底稿（如适用）`
- 面向业务方：使用清晰结论、风险等级、需要业务决策的问题；避免堆砌法条。
- 面向外部或监管：去除内部推理和模型说明；保留必要事实、依据、请求事项和附件清单。
- 不要声称中国法下存在美国式 attorney-client privilege 或 work-product doctrine。可以提示“保密、律师执业秘密、商业秘密或内部管理要求可能适用，需由律师确认具体保护边界”。
