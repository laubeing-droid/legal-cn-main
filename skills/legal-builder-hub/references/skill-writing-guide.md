# claude-for-legal-ZH 法律技能（SKILL.md）编写指南

为了确保 AI 能够精准调用并输出符合中国执业律师标准的法律意见，所有技能文件必须严格执行以下规范：

## 1. 元数据规范 (YAML Frontmatter)
- 文件头部必须由 `---` 包裹。
- `name`: 必须使用小写字母、数字及中划线（`-`），保持全局唯一性。
- `description`: 必须使用 `>` 折叠块语法，且折叠块内的内容每行必须固定**缩进2个空格**。
- `argument-hint`: 必须清晰表述用户在使用该命令时应提供的背景参数。

## 2. 正文格式与工作流
- **唯一一级标题**：正文开头必须是 `# /<插件名>:<技能名>`，作为 AI 识别指令的唯一入口。
- **工作流步骤**：紧随其后必须是 `1.`, `2.`, `3.` 等连续编号的步骤。每个步骤应当是一个明确的"法律工程动作"（如：核查、检索、测算、评估、生成）。
- **二级标题**：必须包含 `## 详细说明`，用于拓展该技能涉及的中国法理逻辑、核心实体条款和核心程序要件。

## 3. 中国法律条文引用要求
- 杜绝模糊引用。禁止出现"根据我国相关法律"、"根据劳动法相关规定"等表述。
- 必须使用**法律全称+精确条款号**。例如：《中华人民共和国公司法（2024年修订）》第23条、《中华人民共和国个人信息保护法》第55条。

## 4. 输出模板代码块规范
- 在技能文件的末尾，必须使用 `### 输出格式示例`，并由 markdown 代码块（反引号）包裹展示给 AI，指定其最终生成的法律文书、合规清单或评估报告的排版版式。
