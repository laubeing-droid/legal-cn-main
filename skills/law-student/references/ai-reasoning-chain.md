## 1.6 AI推理链（System Prompt可用）

以下规则用自然语言+伪代码编写，可直接嵌入Claude System Prompt，不依赖外部工具链。

### 1.6.1 五步推理流水线

处理任何中美法律概念时，按以下顺序逐级推理，不得跳过：

```
Step 1 — 法域识别
  输入: 用户问题或合同条款中涉及的法域信号
  判断: PRC_Mainland | Hong_Kong_SAR | US_Federal | Hybrid（多法域并行）
  规则: 若法域切换（如从美国法→中国法），重置全部映射假设

Step 2 — 阻断检查（优先于映射）
  检查: 该术语是否在22项绝对阻断清单中（见附录5.1）
  若是 → 停止映射，输出阻断说明 + 功能替代方案
  若否 → 继续Step 3

Step 3 — 五级功能判断
  对术语依次回答:
    Q1: 该制度要解决什么问题？（制度目的）
    Q2: 触发后产生什么法律后果？
    Q3: 通过什么程序实现？
    Q4: 哪个机构行使该权力？
    Q5: 违反后果能否强制执行？
  若Q1-Q5均无法建立对应 → 判定为"无对应制度"→ 执行阻断

Step 4 — 标注风险标签
  根据判断结果标注:
    - 完全/部分对应 → 标注对应强度和法系差异提示
    - 功能对应 → 标注替代路径说明
    - 中国独有 → 标注 ★ 中国特色制度
    - 无对应 → 标注 ❌ + 阻断

Step 5 — 合规覆写
  若输出涉及中国强行法（劳动法、刑诉法、数据出境）:
    检查是否隐含了At-Will、Discovery、Plea Bargaining等美式逻辑
    若是 → 强制用中国法逻辑覆写，不可直译
```

### 1.6.2 常见逆向映射场景

**场景A：英文合同中出现 Deed（契据）**

```
判定逻辑:
  IF 文件自称"Deed"且约定了单方义务:
    → 检查是否有对价（consideration）支撑
    → 若以"$1 nominal consideration"满足 → 穿透审查实质
    → 实质为无偿单方承诺 → 中国法定性为赠与合同
    → 警告: 依据《中华人民共和国民法典》第658条，赠与人享有法定任意撤销权
    → 建议: 改为双方互负义务的合同形式，或办理赠与公证
```

**场景B：英文合同中出现 Nominal Consideration（象征性对价）**

```
判定逻辑:
  IF 对价仅为"$1 and other valuable consideration"等象征性表述:
    → 中国法不要求对价，该表述本身无法律效果
    → 关键判断: 合同实质是单务（赠与）还是双务（买卖/服务）
    → 若实质为无偿 → 适用赠与规则（可撤销）
    → 若实质为隐蔽交易 → 依据《中华人民共和国民法典》第146条:
        通谋虚假意思表示无效，按被隐藏的真实法律行为处理
    → 建议: 删除象征性对价条款，改为明确约定各方的实际对待给付
```

### 1.6.3 跨境裁员阻断决策树

```
IF 输入含有"Global RIF"、"terminate without cause"、"severance 2 months":
  → 检测目标实体是否注册于中国大陆
  → IF 是中国大陆实体:
      → 阻断"无理由解雇"逻辑
      → 强制触发以下检查:
          ① 是否满足《中华人民共和国劳动合同法》第41条门槛？
             (裁员>=20人或占总人数10%以上)
          ② 是否提前30天向工会/全体职工说明？
          ③ 是否已向劳动行政部门报告？
      → 若否 → 标注: 违法解除风险，面临2N赔偿或强制恢复劳动关系
      → 话术替换为: "经济性裁员须经法定程序，不可直接通知解雇"
```
  → 边缘情况:
      - 裁员不足20人且不足10% → 不适用第41条，但须符合第39/40条法定解雇事由
      - 涉及孕期/产期/哺乳期女职工 → 第42条绝对禁止经济性裁员
      - 工会缺位 → 仍须向全体职工说明并听取意见，不可跳过
      - 劳务派遣员工 → 退回派遣单位，派遣单位须依法安置或支付补偿
```

### 1.6.4 数据出境触发规则

```
IF 输入涉及"将数据发往境外"、"cloud storage overseas"、"data transfer to parent company":
  → 检查数据类型:
      - 是否包含个人信息？ → 《中华人民共和国个人信息保护法》第38条 触发
      - 是否来自关键信息基础设施？ → 《中华人民共和国网络安全法》第37条 触发
      - 是否属于重要数据？ → 《中华人民共和国数据安全法》第31条 触发
  → 若触发:
      → 标注三轨制处理路径:
          路径A: 安全评估（处理100万人以上/累计出境10万人/敏感1万人）
          路径B: 标准合同备案
          路径C: 个人信息保护认证
      → 警告: 未经评估擅自出境可导致行政处罚+刑事责任
```
  → 边缘/灰区:
      - 匿名化达到"无法识别+不能复原"标准 → 不构成个人信息（个保法第4条）
      - 去标识化但未达匿名化 → 仍属个人信息，出境须走三轨制
      - 境外直接收集境内自然人信息（非从境内转移）→ 个保法第3条第2款仍受管辖
      - 员工个人信息出境至境外母公司 → 须取得员工单独同意或适用其他合法性基础
```
