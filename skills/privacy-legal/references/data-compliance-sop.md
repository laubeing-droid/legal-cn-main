# 数据合规 SOP — AI 可执行版

> 覆盖数据全生命周期：分类分级→收集→存储→使用→共享→出境→删除→应急。
> 来源：`skills/privacy-legal/references/pipil-core-provisions.md` 及配套法规
> 标签：`[个保法第X条]` | `[数安法第X条]` | `[网安法第X条]`

---

## 第一阶段：数据资产盘点（6 项）

- [ ] 1.1 是否已建立数据资产清单？（数据类型/来源/存储位置/处理目的/共享方）
- [ ] 1.2 是否完成数据分类分级？（数安法第21条：一般数据/重要数据/核心数据）
- [ ] 1.3 是否识别出个人信息？（个保法第4条：已识别或可识别自然人的信息）
- [ ] 1.4 是否识别出敏感个人信息？（个保法第28条：生物识别/宗教信仰/医疗健康/金融账户/行踪轨迹/未成年人）
- [ ] 1.5 是否识别出重要数据？（数安法第21条 → 各地区/部门重要数据目录）
- [ ] 1.6 是否识别数据来源合法性？（自行收集/第三方提供/公开爬取）

## 第二阶段：个人信息处理合规（15 项）

### 合法性基础
- [ ] 2.1 处理个人信息是否具备合法性基础？（个保法第13条七选一）
- [ ] 2.2 如基于"同意"：是否取得清晰、自愿、明确的同意？（个保法第14条）
- [ ] 2.3 处理目的/方式/种类变更是否重新取得同意？（个保法第14条第2款）
- [ ] 2.4 撤回同意是否与给予同意同等便捷？（个保法第15条）
- [ ] 2.5 处理不满14周岁未成年人信息是否取得监护人同意？（个保法第31条）

### 告知义务
- [ ] 2.6 是否提供隐私政策/个人信息处理规则？（个保法第17条）
- [ ] 2.7 隐私政策是否包含：处理者名称/联系方式+处理目的/方式+信息种类/保存期限+个人行使权利的方式？（个保法第17条）
- [ ] 2.8 隐私政策是否以显著方式、清晰易懂的语言呈现？（不得隐藏在冗长用户协议中）

### 敏感个人信息
- [ ] 2.9 处理敏感个人信息是否取得单独同意？（个保法第29条）→ 不得一揽子同意
- [ ] 2.10 是否告知处理敏感个人信息的必要性及对个人的影响？（个保法第30条）
- [ ] 2.11 是否进行了个人信息保护影响评估（PIA）？（个保法第55条：敏感个人信息/自动化决策/委托处理/出境/公开）

### 自动化决策
- [ ] 2.12 是否利用个人信息进行自动化决策？（如用户画像、算法推荐）→ 需保证透明度+结果公平
- [ ] 2.13 自动化决策是否向个人提供不针对个人特征的选项或拒绝方式？（个保法第24条）
- [ ] 2.14 是否存在"大数据杀熟"？（交易价格等交易条件上不合理差别待遇）→ 违法（个保法第24条第1款）
- [ ] 2.15 AI生成内容是否依法标识？（深度合成管理规定第17条/AI内容标识办法）

## 第三阶段：数据安全保护措施（8 项）

- [ ] 3.1 是否指定个人信息保护负责人？（个保法第52条：处理100万人以上/敏感个人信息 → 必须）
- [ ] 3.2 是否建立数据安全管理制度和操作规程？（数安法第27条/网安法第21条）
- [ ] 3.3 是否采取加密、去标识化、匿名化等技术措施？（数安法第27条）
- [ ] 3.4 是否对数据处理活动进行日志记录？（网安法第21条第3项：不少于6个月）
- [ ] 3.5 是否进行了数据安全风险评估？（数安法第22条）
- [ ] 3.6 是否建立数据安全事件应急预案？（数安法第23条/网安法第25条）
- [ ] 3.7 重要数据处理者是否定期开展风险评估并报送？（数安法第30条）
- [ ] 3.8 是否对数据接收方/受托方进行数据安全能力评估？（个保法第21条）

## 第四阶段：数据委托处理与共享（6 项）

- [ ] 4.1 委托处理是否签订数据处理协议？（个保法第21条：约定处理目的/方式/种类/保护措施）
- [ ] 4.2 受托方是否超出约定范围处理个人信息？→ 违法
- [ ] 4.3 委托关系解除后受托方是否返还或删除数据？（个保法第21条第3款）
- [ ] 4.4 向其他处理者提供个人信息是否取得单独同意？（个保法第23条）
- [ ] 4.5 合并/分立/解散/破产时接收方是否继续履行保护义务？（个保法第22条）
- [ ] 4.6 是否向境外提供个人信息？（个保法第38条）→ 进入第五阶段

## 第五阶段：数据出境（8 项）

- [ ] 5.1 是否进行了数据出境安全评估？（出境安全评估办法第4条：重要数据/100万人+/累计10万敏感/1万敏感→必须）
- [ ] 5.2 如不满足安全评估条件：是否签署了个人信息出境标准合同（SCC）并向省级网信办备案？
- [ ] 5.3 是否取得了个人信息保护认证？（个保法第38条三选一：安全评估/SCC/认证）
- [ ] 5.4 是否取得个人的单独同意并告知出境接收方信息？（个保法第39条）
- [ ] 5.5 是否完成了个人信息保护影响评估（PIA）？（个保法第55条）
- [ ] 5.6 出境接收方所在国/地区的数据保护水平是否评估？（个保法第38条）
- [ ] 5.7 是否涉及禁止出境的数据类型？（数据出境安全评估办法第2条：国家秘密/核心数据/特定行业）
- [ ] 5.8 是否属于境外司法/执法机构调取数据情形？（个保法第41条：非经主管机关批准不得提供）

## 第六阶段：数据主体权利响应（7 项）

- [ ] 6.1 是否建立个人信息权利请求受理机制？（个保法第50条）
- [ ] 6.2 查阅/复制权：是否能及时提供个人信息的查阅/复制？
- [ ] 6.3 更正权：是否能及时更正不准确/不完整的个人信息？
- [ ] 6.4 删除权：是否满足删除条件及时删除？（个保法第47条：目的实现/撤回同意/违法处理等）
- [ ] 6.5 可携带权：是否支持个人信息转移至其他处理者？（个保法第45条第3款，需网信办规定）
- [ ] 6.6 被遗忘权/限缩：是否支持"近亲属为自身合法正当利益可行使死者信息权利"？（个保法第49条）
- [ ] 6.7 是否向个人提供便捷的投诉渠道？（个保法第50条第2款）

## 第七阶段：应急响应（5 项）

- [ ] 7.1 发生数据泄露/安全事件是否立即启动应急预案？（网安法第25条）
- [ ] 7.2 是否在72小时内通知履行数据安全保护职责的部门？（网安法第25条——但个保法要求"立即"，建议24小时内）
- [ ] 7.3 是否及时通知受影响的个人？（个保法第57条：告知事件情况+可能后果+已采取措施+个人应对建议）
- [ ] 7.4 是否采取措施防止危害扩大？（网安法第25条）
- [ ] 7.5 事件处置后是否进行复盘并更新防护措施？

---

> **总计：55 项检查点** | 覆盖个保法 + 数安法 + 网安法 + 配套规定
