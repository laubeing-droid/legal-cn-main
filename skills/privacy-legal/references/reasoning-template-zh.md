# 数据/隐私中国法推理模板

> 用途：替代 GDPR / CCPA / U.S. sectoral privacy laws 等域外推理

## 禁用的美国法推理模式

- ❌ CCPA opt-out framing — 中国以"告知+同意"为核心
- ❌ GDPR legitimate interest balancing — 中国合法性基础清单不同
- ❌ U.S. sectoral approach (HIPAA/FCRA/GLBA/COPPA) — 中国为统一三法框架
- ❌ Privacy Shield / SCC 跨境机制 — 中国有独立的三轨制出境体系

## 中国法推理链

### 三法定位
```
个人信息保护法（PIPL）：个人信息全生命周期
数据安全法（DSL）：数据分类分级 + 安全审查 + 出口管制
网络安全法（CSL）：网络运营者安全义务 + 关键信息基础设施保护（CII）
```

### 数据处理合规分析
```
Step 1 — 合法性基础
  → 是否取得个人同意？（PIPL第13条）
  → 同意是否"充分知情+自愿+明确"？
  → 是否属于第13条其他6种合法性基础？（合同必要/法定义务/紧急避险等）

Step 2 — 告知义务
  → 处理前是否告知处理目的/方式/种类/保存期限？（PIPL第17条）
  → 敏感个人信息是否额外告知必要性+对个人权益的影响？（第30条）

Step 3 — 敏感个人信息
  → 是否属于敏感信息？（生物识别/宗教信仰/特定身份/医疗健康/金融账户/行踪轨迹/不满14周岁未成年人信息）
  → 是否取得单独同意？（第29条）
  → 是否有特定目的+充分必要性？（第28条）

Step 4 — 数据出境三轨制
  → 路径A：安全评估（处理100万人以上/累计出境10万人/敏感1万人）
  → 路径B：标准合同备案
  → 路径C：个人信息保护认证
```

### 数据安全审查
```
Step 1 — 是否构成CIIO（关键信息基础设施运营者）？
  → 若为CIIO → 境内存储+出境安全评估（CSL第37条）
Step 2 — 是否需要进行数据安全审查？
  → 影响/可能影响国家安全的数据处理活动（DSL第24条）
Step 3 — 数据分类分级
  → 一般数据/重要数据/核心数据（DSL第21条）
```
